Example

Representative migration pattern

This example illustrates the intended maintenance scenario: AST-based transforms, API migrations, and project-wide automation for Python codebases. 

from pymorph import project, find_files, transform_file

for file in find_files("src/**/*.py"):
    transform_file(file, lambda source: (
        project(source)
        .rename_import("legacy.config", "core.settings")
        .rename_symbol("LegacyConfig", "ProjectConfig")
        .to_source()
    ))

API design

Small primitives for large Python codebase changes

The package story is now fully centered on Python code processing. The API names suggest parsing, selection, transformation, emission, and project-wide execution instead of UI rendering.

parse()

Convert Python source into an AST with version-aware parser settings for migrations and analysis.

select()

Match Python syntax nodes with a concise selector layer for refactors, lint rules, and analysis.

replace()

Rewrite targeted syntax with deterministic AST transforms rather than fragile text replacement.

emit()

Generate formatted Python source from transformed trees while preserving clean readable output.
